# Унифицированный сервер для Сервиса Электронного Подписания
Платформа: Net Core 9
БД: Posgre
Не используем Entity Framework
Обязательно : чистая архитектура
Подход: Domain Driven + TDD

## Описание
Отдельно стоящая серверная часть для вспомогательного функционала.

! Важно ! В дальнейшем сервер будет расширять свой функционал

Весь функционал разделен на модули, которые можно считать либо плагинами либо микросервисами. В зависимости от выбора итоговой архитектуры

## Субсервис 1

Сервис аутентификации, авторизации - простейшая авторизация для обеспечения безопасности данных

## Субсервис 2

Субсервис обработки pdf с помощью локальной ML модели. Выделение авторов с должностями и всех кто должен подписать чертеж из штампа.

### Описание

1) Субсервис получает hash файла, если есть в БД результатом обращения получаем json обработки, если нет - возврат отсутствия данных.
2) Субсервис получает файл, если по хэш что-то найдено в БД - возвращаем json. Если не найдено, поднимается модель и обрабатывает файл - разбор PDF поиск фамилий и должностей. Возможны файлы более 100Мб. 
!Важно: Обработка может длиться даже более 5 минут. Надо продумать маштабирование. Чтоб одновременно обрабатывать множество запросов. Сейчас развернуто как Rest API - 1 метод получает файл возвращает json:
https://pdf2txt.selectel.dt1520.ru/recognize_json/
