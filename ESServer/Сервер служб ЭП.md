# Сервер служб ЭП

Стартовая страница Obsidian-базы по проекту серверной платформы для сервиса электронного подписания.

Проект описывает не один узкий REST-сервис, а расширяемую серверную платформу, которая предоставляет desktop-приложению сервиса ЭП набор серверных возможностей: обработку PDF, обработку изображений, вызов внешних или лицензированных сервисов, хранение результатов по hash, администрирование поставки и эксплуатационный контроль.

## Коротко

- Платформа: .NET 9.
- База данных: PostgreSQL.
- Entity Framework не используем.
- Архитектура: чистая архитектура, DDD, TDD.
- Основная форма: модульный монолит по коду, несколько процессов по эксплуатации.
- Первый важный сценарий: асинхронная обработка PDF через внешний ML-сервис, который считается чёрным ящиком.
- Админка входит в MVP и должна быть качественной, понятной прикладному администратору.

## Принцип проекта одной фразой

Сервис ЭП обращается не к конкретным ML-сервисам или endpoint-ам, а к понятным бизнес-возможностям платформы; сервер сам выбирает настроенный обработчик, ставит задачу в очередь, контролирует повторные попытки и отдаёт результат по hash/job.
