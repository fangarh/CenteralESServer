## Админка

- Нужен ли экспорт отчёта по job/result для передачи в поддержку?

Уже согласовано:

- основные действия: упавшие задачи, retry, проверки работоспособности, API keys, поставка компонентов;
- Dashboard гибридный: общая сводка + рабочая консоль при проблемах;
- runtime-параметры компонента управляются на странице компонента;
- опасные изменения требуют подтверждения;
- retry одной задачи и массовый retry по фильтру входят в MVP;
- проверки делятся на Basic health, Processor health и Optional diagnostic test;
- админка не запускает обычную бизнес-обработку файлов.
- отдельный экран “Поставка” входит в MVP и показывает состав текущей установки.
- глобальный DevOps mode в MVP не нужен; технические детали доступны в раскрываемых блоках на конкретных страницах.
